## 需求 3: 架构基建的双向同步引擎 (Bidirectional Sync Engine)

**日期**: 2026-04-16
**状态**: ✅ 已实施
**提出者**: User

**意图**:
1. 解决 Onebot 提取为独立仓库后，如何进行迭代优化的问题。
2. 解决 Onebot 迭代优化后，如何安全地更新到下游的业务项目（如 claw-swarm）的问题。
3. 必须支持“业务驱动，就地演进”的哲学（即在下游业务项目中修改基建，验证后反向推送回母体仓库）。
4. 向下同步时，必须保护下游项目的业务级配置（如 `opencode.json` 中的业务命令和业务 `INTENTS.md`）。

**期望结果**: 
- 提供一种纯净的源码拉取与反哺机制（非 Git Submodule 嵌套方案）。
- 实现一键式的 CLI 工具（如 `onebot.sh`）。

**关联 Spec**: `.opencode/tools/sync-engine/SPEC.md`
